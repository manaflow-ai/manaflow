## Credentials and local requirements

cmux checks requirements before you start a run. Use Settings to add keys, and check Provider Status to see missing requirements.

### Claude Code
- Settings keys: `CLAUDE_CODE_OAUTH_TOKEN` or `ANTHROPIC_API_KEY`.
- If no key is set, cmux looks for `~/.claude.json` plus `~/.claude/.credentials.json` or macOS keychain entries named "Claude Code" or "Claude Code-credentials".

### Codex CLI
- Settings keys: `CODEX_AUTH_JSON` (preferred) or `OPENAI_API_KEY`.
- `CODEX_AUTH_JSON` should be the full contents of `~/.codex/auth.json`.
- Local checks require `~/.codex/auth.json` and `~/.codex/config.toml` to exist.

### Gemini CLI
- Requires `~/.gemini/settings.json`.
- Auth via `GEMINI_API_KEY` or OAuth files in `~/.gemini` (`oauth_creds.json`, `google_accounts.json`, `google_account_id`).

### Amp
- Settings key: `AMP_API_KEY`.
- Or local `~/.local/share/amp/secrets.json`.

### Cursor
- Settings key: `CURSOR_API_KEY`.
- Or local `~/.config/cursor/auth.json` or macOS keychain tokens `cursor-access-token` and `cursor-refresh-token`.
- Requires `~/.cursor/cli-config.json`.

### Opencode
- Requires local `~/.local/share/opencode/auth.json`.
- Provider keys by model:
  - `ANTHROPIC_API_KEY`: opencode/sonnet-4, opencode/opus-4, opencode/opus-4.1-20250805, opencode/qwen3-coder
  - `OPENAI_API_KEY`: opencode/o3-pro, opencode/gpt-5, opencode/gpt-5-mini, opencode/gpt-5-nano
  - `OPENROUTER_API_KEY`: opencode/kimi-k2, opencode/glm-4.5, opencode/gpt-oss-120b, opencode/gpt-oss-20b
  - `XAI_API_KEY`: opencode/grok-4-1-fast, opencode/grok-4-1-fast-non-reasoning
  - opencode/grok-code does not need a provider key but still requires the opencode auth file.

### Qwen Code
- `qwen/qwen3-coder:free` uses `OPENROUTER_API_KEY`.
- `qwen/qwen3-coder-plus` uses `MODEL_STUDIO_API_KEY`.
- Both keys are injected as `OPENAI_API_KEY` for the CLI.

## Selecting agents
- You can pick multiple agents for a single task.
- You can add multiple instances of the same agent. The picker caps instances at 6 per agent.
- If an agent is unavailable, the picker shows the missing requirement and prevents selection.
